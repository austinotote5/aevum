# AEVUM 14-Day Focus Freeze

## Window
- Start: March 26, 2026
- End: April 9, 2026

## Objective
Raise technical maturity and trust posture by freezing non-essential feature work.

## Allowed Work
1. Reliability defects (route failures, session/auth instability, error handling gaps)
2. Security and compliance hardening
3. Test coverage and CI quality gates
4. Operational runbooks and rollback readiness
5. Claims/safety governance and auditability improvements

## Blocked During Freeze
1. Net-new product surfaces unrelated to reliability/compliance
2. Cosmetic-only UI expansions
3. New data simulations or synthetic growth shortcuts

## Exit Criteria (April 9, 2026)
1. API contract suite passing
2. CI gate required for merge/release
3. Incident, DB restore, and rollback runbooks complete
4. Claims boundary and safety escalation policies approved
5. Real-data proof pipeline validated (no synthetic/demo contamination)
